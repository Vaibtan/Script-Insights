Script Insights Agentic Architecture

Presentation view of the v1 system across client experience, FastAPI orchestration, agent execution, durable persistence, and optional DSPy plus Groq inference.

Client

  • - Next.js provides the workspace plus run, history, and compare views.
  • - Text and PDF ingestion share the same client entry point.
  • - Query state and polling stay in the frontend layer.

Execution

  • - FastAPI routes dispatch into submission, query, and background execution paths.
  • - The workflow normalizes input, runs five agents, and applies guardrails.
  • - DSPy can call Groq, but deterministic fallbacks always exist.

State

  • - SQLAlchemy hides persistence mapping behind a gateway and codec layer.
  • - SQLite stores run history, revisions, artifacts, normalized scripts, and sources.
  • - Structured logs and request IDs support traceability across the platform.
